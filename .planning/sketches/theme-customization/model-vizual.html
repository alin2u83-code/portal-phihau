③ Înainte vs După — cum se schimbă codul
⚠️
Problema actuală: Culorile sunt hardcodate Tailwind (bg-slate-800, text-blue-600) în fiecare componentă. Schimbi tema = schimbi 100+ fișiere. Imposibil de întreținut.
❌ Înainte — culori hardcodate
/* Sidebar.tsx */ <div className="bg-slate-800 text-slate-300"> <nav className="bg-blue-600 text-white"> /* item activ */ </nav> </div> /* Table.tsx */ <thead className="bg-slate-100 text-slate-600"> <tr className="even:bg-slate-50 hover:bg-blue-50"> /* Button.tsx */ <button className="bg-blue-600 hover:bg-blue-700"> → Tema nouă = modific 80+ fișiere → Imposibil consistent
→
✅ După — tokenuri CSS
/* Sidebar.tsx */ <div style={{background:'var(--sb-bg)', color:'var(--sb-text)'}}> <nav style={{background:'var(--sb-active)'}}> </nav> </div> /* Table.tsx */ <thead style={{background:'var(--tbl-header)'}}> /* CSS: tr:nth-child(even){background:var(--tbl-stripe)} */ /* Button.tsx */ <button style={{background:'var(--accent)'}}> → Tema nouă = 1 funcție applyTheme() → Tot se schimbă instant
④ Unde se stochează tema
⚡
CSS Custom Properties
Stare live pe document.documentElement. Se aplică instant. Se pierde la refresh dacă nu e salvată. Sursa de adevăr pentru browser.
Runtime
📦
localStorage
JSON cu toate tokenurile. Se citește la startup și se aplică înainte de primul render. Supraviețuiește refresh și restart browser.
Persistent client
☁️
Supabase (opțional)
Coloana tema_preferata pe tabelul utilizatori. Sincronizează tema pe toate dispozitivele utilizatorului. Necesită migrație DB.

FOUC (Flash of Unstyled Content) — dacă nu setezi CSS vars înainte de React mount, utilizatorul vede 1 frame cu tema default înainte de tema lui. Rezolvare: <script> inline în <head> din index.html (nu în React).
⑥ Ce rezolvă automat din audit Impeccable
❌ Probleme actuale (Impeccable)
48× gray-on-color — WCAG AAA fail
border-l-4 hardcodat în 30 fișiere
Nicio scalabilitate — culori ad-hoc
Dark mode imposibil fără refactor
✅ Ce rezolvă token system
Preset-urile au contrast calculat — zero gray-on-color
border-l-4 → var(--accent) + 1 loc
Dark mode = un preset nou, zero refactor
Fiecare temă nouă = ~20 linii JSON
💡
Pașii de implementare:
1. Definești tokenurile în themeTokens.ts (obiect cu cheile CSS var)
2. Funcție applyTheme(tokens) → setează pe document.root
3. Zustand store useThemeStore → apelează applyTheme la schimbare
4. Script inline în index.html → citește localStorage și aplică tema la startup
5. Înlocuiești Tailwind hardcodat cu style={{background:'var(--sb-bg)'}} progresiv — nu totul odată
