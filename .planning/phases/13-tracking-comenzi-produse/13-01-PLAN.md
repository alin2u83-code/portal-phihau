    Tabele (schema completă din RESEARCH.md liniile 380-444 și PATTERNS.md liniile 49-78):
    1. `comenzi_produse`: id, club_id (FK cluburi ON DELETE CASCADE — **NULLABLE**, vezi DECIZIE club_id mai jos), tip_comanda (NOT NULL), stare (DEFAULT 'DESCHISA'), furnizor TEXT, observatii TEXT, created_by UUID, created_at, updated_at.

    **DECIZIE club_id pe comenzi_produse (rezolvă incompatibilitatea cu Flux B `federatie_club`):** `club_id` este **NULLABLE** (NU NOT NULL), cu un CHECK constraint care impune prezența lui doar pentru fluxul club→furnizor. Adaugă explicit în CREATE TABLE:
    `CONSTRAINT comenzi_produse_club_id_chk CHECK ((tip_comanda = 'club_furnizor' AND club_id IS NOT NULL) OR (tip_comanda IN ('federatie_club', 'club_federatie')))`.
    Motivare: pentru `federatie_club` comanda este inițiată de federație și nu aparține unui club anume (destinatarii reali sunt în `comenzi_produse_cluburi`); pentru `club_federatie` clubul sursă POATE seta club_id (clubul inițiator) dar nu e obligatoriu. Aceasta este soluția cea mai simplă — fără coloană `federatie_id` separată. Service-ul (13-04 `creareComandaFederatie`) lasă `club_id = null` pentru `federatie_club`. Accesul clubului destinatar la comenzile federației trece prin `comenzi_produse_cluburi.club_id`, nu prin `comenzi_produse.club_id`.
    2. `cereri_produse`: id, club_id (FK cluburi CASCADE), sportiv_id (FK sportivi ON DELETE SET NULL), comanda_id (FK comenzi_produse ON DELETE SET NULL), varianta_id (FK produse_variante NOT NULL), cantitate INTEGER DEFAULT 1 CHECK (>0), stare_cerere (DEFAULT 'SOLICITATA'), platit_dupa_predare BOOLEAN DEFAULT false, plata_id (FK plati ON DELETE SET NULL), batch_urmatoarea BOOLEAN DEFAULT false, observatii TEXT, created_by UUID, created_at, updated_at.
    3. `comenzi_produse_iteme`: id, comanda_id (FK comenzi_produse CASCADE), varianta_id (FK produse_variante), cantitate INTEGER CHECK (>0), created_at.
    4. `comenzi_produse_cluburi`: id, comanda_id (FK comenzi_produse CASCADE), club_id (FK cluburi CASCADE), cantitate INTEGER CHECK (>0), confirmat BOOLEAN DEFAULT false, confirmat_at TIMESTAMPTZ, created_at, UNIQUE (comanda_id, club_id).

    ALTER tabela existentă (per CMD-09, guard idempotent — Assumption A1 RESEARCH.md):
    `ALTER TABLE produse ADD COLUMN IF NOT EXISTS tip_produs TEXT NOT NULL DEFAULT 'per_sportiv' CHECK (tip_produs IN ('per_sportiv', 'per_club'));`

    RLS — ENABLE ROW LEVEL SECURITY pe toate cele 4 tabele noi. Policies copiate exact din pattern PATTERNS.md liniile 83-103, folosind public.is_super_admin() și public.get_active_club_id():
    - cereri_produse SELECT: is_super_admin() OR club_id = get_active_club_id() OR sportiv_id IN (SELECT id FROM sportivi WHERE user_id = auth.uid())
    - cereri_produse INSERT: is_super_admin() OR club_id = get_active_club_id() OR sportiv_id IN (SELECT id FROM sportivi WHERE user_id = auth.uid() AND club_id = cereri_produse.club_id)
    - cereri_produse UPDATE: is_super_admin() OR club_id = get_active_club_id()
    - comenzi_produse SELECT/INSERT/UPDATE: is_super_admin() OR club_id = get_active_club_id() OR EXISTS (SELECT 1 FROM comenzi_produse_cluburi cpc WHERE cpc.comanda_id = comenzi_produse.id AND cpc.club_id = get_active_club_id()). **Notă club_id NULLABLE:** comparația `club_id = get_active_club_id()` returnează NULL (deci fals) când club_id este NULL — corect; comenzile federației (club_id NULL) sunt accesibile prin is_super_admin() pentru federație și prin join-ul comenzi_produse_cluburi pentru clubul destinatar.
    - comenzi_produse_iteme SELECT/INSERT/UPDATE: is_super_admin() OR EXISTS join pe comanda_id → comenzi_produse (club_id = get_active_club_id() OR comanda are destinatar comenzi_produse_cluburi cu club_id curent).
    - comenzi_produse_cluburi SELECT/INSERT/UPDATE: is_super_admin() OR club_id = get_active_club_id() (clubul destinatar vede/confirmă propria recepție).

    NU folosi fenced code în alte locuri — scrie SQL valid direct în fișier. Toate CREATE TYPE / CREATE TABLE / CREATE POLICY trebuie idempotente (IF NOT EXISTS sau DROP POLICY IF EXISTS înainte de CREATE POLICY).
