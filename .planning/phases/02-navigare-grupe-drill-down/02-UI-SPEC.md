## Interaction Contracts

### GrupaCard — Button Layout (Post D-07)

```
[Detalii]       [Gestionează]       [...]
 primary (indigo)  secondary (slate)  secondary (slate)
```

- "Detalii" click → `onDetalii(grupa)` → `setGrupaSelectedForDetail(grupa)` in parent → renders `GrupaDetailView`
- "Gestionează" click → `onEdit(grupa)` → opens `GrupaFormModal` (unchanged)
- "..." click → opens dropdown with: Modifică Program | Generează Antrenamente | Secundari | separator | Șterge Grupa

### GrupaDetailView — View Swap

- Triggered by: `grupaSelectedForDetail !== null` in `Grupe/index.tsx`
- Render method: conditional render (`{grupaSelectedForDetail ? <GrupaDetailView ...> : <gridCarduri>}`) — NOT NavigationContext view swap (per D-01 intent: state local în Grupe/index.tsx; planner confirms per code_context line 77)
- Back navigation: "← Înapoi la Grupe" button at top-left → `setGrupaSelectedForDetail(null)` + hardware back button via NavigationContext history stack

### Tab Bar

- 3 tabs rendered as `<button>` elements in a horizontal strip
- Active tab: `border-b-2 border-indigo-500 text-white font-semibold` 
- Inactive tab: `text-slate-400 hover:text-slate-200 border-b-2 border-transparent`
- Default active tab on mount: **Antrenamente** (per D-09)
- Tab state: local `useState<'antrenamente' | 'orar' | 'sportivi'>('antrenamente')` in `GrupaDetailView`
- Tab persistence: preserved while `GrupaDetailView` is mounted; NOT persisted across Back/forward navigation (Phase 2 MVP)

### Tab Antrenamente (Phase 2 Placeholder — D-10)

- Displays: empty state card with placeholder copy (see Copywriting section)
- No query, no calendar, no actions
- Visual: centered in tab content area, `border-dashed border-2 border-slate-700 rounded-xl py-12`

### Tab Orar (D-02, D-03, D-04)

- Inline copy of `OrarEditorModal` logic without `<Modal>` wrapper
- Layout: full-width within tab content area (no max-width constraint from Modal's `max-w-2xl`)
- "Anulează" button behavior: resets local `program` state to `grupa.program` original (Undo) — does NOT navigate away
- "Salvează Orar" button behavior: saves to Supabase → invalidates React Query cache → shows success toast → stays on Tab Orar (per D-04)
- Save button variant: `success` (emerald-600)
- Cancel button label: "Resetează" (not "Anulează" — to avoid confusion with a close action in a no-modal context)

### Tab Sportivi (D-05, D-06)

- Read-only list: each row = `{prenume} {nume}` + `{grad_actual}` in `text-slate-300 text-sm`
- Query: separate Supabase query for sportivi per grupă (not from DataContext)
- "Adaugă Sportivi" button: variant `info` (sky-600), opens `AdaugaSportiviModal` as modal (existing behavior unchanged)
- No remove/action per sportiv (Phase 2 = read-only per D-06)

## States Required

Each new UI surface must implement these states:

### GrupaDetailView

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading | While `grupaSelectedForDetail` hydrates | Full-area spinner centered |
| Default | Loaded, tab active | Normal content per active tab |

### Tab Orar

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Idle | No pending changes | "Resetează" disabled or visually dim; "Salvează Orar" enabled |
| Dirty | User made a change | Both buttons enabled |
| Saving | After "Salvează Orar" click | Button shows `isLoading` spinner, inputs disabled |
| Save success | After Supabase confirms | Success toast (react-hot-toast), state resets to saved version |
| Save error | Supabase returns error | Error toast via `showError`, state NOT reset (user keeps edits) |
| Empty (no intervals for a day) | Day has 0 intervals | Dashed border clickable zone: "Niciun antrenament pentru {zi}. Apasă pentru a adăuga primul interval." |

### Tab Sportivi

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading | Query in flight | Row skeleton or spinner |
| Empty | Query returns 0 sportivi | Empty state card (see Copywriting) |
| Populated | Query returns sportivi | Read-only list rows |
| Error | Query fails | Error state with retry link |
