  Creează fișier cu migrație idempotentă. Structura:

  ```sql
  -- Migrație: Adaugă tema_config (jsonb) pe cluburi și utilizatori
  -- Sprint 1: Color Theme System
  -- Idempotentă: poate fi rulată de mai multe ori fără erori

  -- 1. Coloana tema_config pe cluburi (separată de theme_config existent)
  ALTER TABLE cluburi
    ADD COLUMN IF NOT EXISTS tema_config jsonb DEFAULT NULL;

  -- 2. Coloana tema_config pe utilizatori
  ALTER TABLE utilizatori
    ADD COLUMN IF NOT EXISTS tema_config jsonb DEFAULT NULL;

  -- 3. RLS: orice user autentificat poate citi tema clubului său
  -- (deja acoperit de politica SELECT existentă pe cluburi — verifici prin: SELECT policyname FROM pg_policies WHERE tablename='cluburi')

  -- 4. RLS: ADMIN_CLUB poate actualiza cluburi.tema_config pentru clubul propriu
  -- Politica existentă de UPDATE pe cluburi trebuie să permită column-level access pe tema_config
  -- Dacă politica existentă e pe tot rândul (row-level fără column restriction), funcționează automat.
  -- Dacă e nevoie de politică specifică:
  DO $$
  BEGIN
    IF NOT EXISTS (
      SELECT 1 FROM pg_policies
      WHERE tablename = 'cluburi' AND policyname = 'cluburi_tema_config_admin_update'
    ) THEN
      CREATE POLICY cluburi_tema_config_admin_update
        ON cluburi
        FOR UPDATE
        USING (
          EXISTS (
            SELECT 1 FROM utilizator_roluri_multicont urm
            WHERE urm.user_id = auth.uid()
              AND urm.club_id = cluburi.id
              AND urm.rol_denumire IN ('ADMIN_CLUB', 'SUPER_ADMIN_FEDERATIE')
          )
        );
    END IF;
  END $$;

  -- 5. RLS: orice user autentificat poate citi și actualiza propria linie din utilizatori
  DO $$
  BEGIN
    IF NOT EXISTS (
      SELECT 1 FROM pg_policies
      WHERE tablename = 'utilizatori' AND policyname = 'utilizatori_tema_config_self_update'
    ) THEN
      CREATE POLICY utilizatori_tema_config_self_update
        ON utilizatori
        FOR UPDATE
        USING (auth.uid() = user_id)
        WITH CHECK (auth.uid() = user_id);
    END IF;
  END $$;
  ```

  NOTĂ pentru executor: Rulează acest SQL în Supabase SQL Editor sau via Supabase CLI (`supabase db push`). Verifică după rulare că coloanele există cu query-ul din acceptance_criteria.
