3 abordări pentru sistemul de teme

Alege abordarea arhitecturală — fiecare are alt raport efort/acoperire

A

Minimal — CSS vars pe componente cheie

12 variabile CSS semantice în :root. Migrezi doar Button + Card din ui.tsx. Restul aplicației rămâne cu Tailwind hardcodat.

Acoperire temă
~60% din UI
Efort implementare
2–3 zile

Pro

  • Rapid de livrat
  • Risc minim de regresii
  • Butoanele și cardurile urmează tema

Contra

  • Sidebar, tabele, badge-uri rămân hardcodate
  • Experiență inconsistentă
B

Complet — migrare întreagă ui.tsx

Aceleași CSS vars, dar migrezi toate cele ~20 componente din ui.tsx: Button, Card, Modal, Badge, Input, Table, Alert, Tabs, etc.

Acoperire temă
~85% din UI
Efort implementare
5–7 zile

Pro

  • Acoperire completă fără atingerea paginilor
  • Paginile folosesc automat noile variabile
  • Un singur fișier de migrat: ui.tsx

Contra

  • Risc de regresii vizuale în ui.tsx
  • Pagini custom (Competitii, Plati) cu clase inline rămân în afara temei
C

✦ Recomandat — Fazat (A acum, B în sprint 2)

Livrezi rapid: ThemeEditor + CSS vars + Button tema-aware. Sprint 2: extinzi la restul ui.tsx. Utilizatorii văd schimbarea imediat fără a aștepta migrarea completă.

Acoperire sprint 1
60% → 85% după sprint 2
Time-to-value
2–3 zile pentru sprint 1
Sprint 1 (acum)
  • ThemeContext + CSS vars + ThemeEditor modal
  • Migrare Button (toate variantele urmează tema)
  • 8 teme predefinite + editor culori custom
  • Supabase: coloane tema_config pe cluburi + role_contexts
Sprint 2 (după)
  • Migrare Card, Modal, Badge, Input, Table, Tabs, Alert
  • Acoperire 85%+ fără atingerea paginilor
