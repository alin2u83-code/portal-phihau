## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| useCallback in non-hook factory | createSetter called useCallback inside regular function — Rules of Hooks violation (CR-03) | advisory | Factory functions must not call Hook APIs; setData from useState is stable |
| Insert order matters for atomicity | rezultate inserted before plati meant no rollback path on partial failure (CR-01) | advisory | Always insert payment (plati) first; compensating delete on plati failure before throwing |
| Loose join fallback contamination | sportiv_id-only fallback in platiParticipanti matched payments from other events (CR-02) | advisory | Strict match on eveniment_id only; never add loose fallback for DB joins |
