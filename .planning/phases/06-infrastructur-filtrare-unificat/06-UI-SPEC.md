## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Icon-to-text gap inside toggle button |
| sm | 8px (`gap-2`, `p-2`) | Gap between filter chips; gap between range inputs and dash separator |
| md | 12px (`p-3`, `gap-3`) | Filter panel internal padding; grid gap between filter groups |
| lg | 16px (`mb-2`, `space-y-3`) | Space between toggle button row and filter panel |
| xl | 24px (`space-y-6`) | Outer container spacing (inherited from parent tab layout) |

Exceptions:

- Toggle button touch target: `px-3 py-1.5` (12px / 6px) — intentionally compact, inline button not a primary CTA
- Age/grade range inputs: `w-20` (80px) — fixed narrow width to keep two inputs on one line
- Section labels: `text-[11px]` — 11px tracking-wide all-caps, below the 12px body floor, used only for filter group headers (4 occurrences max)

## Component Inventory

### CompetitieFilterBar

**File:** `components/Competitii/CompetitieFilterBar.tsx`

**Anatomy (top to bottom):**

```
[wrapper div]
  [row: toggle button + optional reset link]
    [toggle button]
      [filter SVG icon 14×14]
      ["Filtrează" or "Filtrează (N)"]
      [chevron SVG 12×12 — rotates 180° when open]
    [reset link — visible only when nrFiltreActive > 0]
      ["Reset" — text-xs text-slate-400 hover:text-white underline]

  [collapsible panel — visible only when filtreVisible === true]
    [2-column responsive grid (1col mobile / 2col md+)]
      [Gen group]
        [label "GEN"]
        [chip row: "Feminin" | "Masculin" | "Mixt"]
          each chip = <label> wrapping hidden <input type="checkbox">
      [Probă group]
        [label "PROBĂ"]
        [native <select> with "Toate probele" + probe[] options]
      [Vârstă group]
        [label "VÂRSTĂ (ANI)"]
        [<input min> — 80px] [dash "–"] [<input max> — 80px]
      [Grad group]
        [label "GRAD"]
        [native <select gradMin> "Orice grad" + grade[] options]
        [dash "–"]
        [native <select gradMax> "Orice grad" + grade[] options]
```

**States:**

| State | Visual |
|-------|--------|
| Collapsed, no active filters | Toggle button: `bg-slate-800 border-slate-700 text-slate-300`; chevron points down |
| Collapsed, filters active | Toggle button: `bg-brand-primary/20 border-brand-primary/50 text-brand-primary`; badge "(N)" visible |
| Expanded | Panel visible; chevron rotates 180° (`rotate-180` + `transition-transform`) |
| Chip inactive | `bg-slate-700/50 border-slate-600 text-slate-300 hover:bg-slate-700` |
| Chip active | `bg-brand-primary/20 border-brand-primary/50 text-brand-primary` |
| Input focused | `focus:border-brand-primary/60 focus:outline-none` |
| Select | `bg-slate-700 border-slate-600 text-white` — native OS styling for options |

**Internal state (only allowed state in this component):**

- `const [filtreVisible, setFiltreVisible] = useState(false)` — initial value: `false` (collapsed at mount)

**Source for initial value decision:** RESEARCH.md Assumption A2 — collapsed is correct default UX because 4 tabs will each mount their own FilterBar; expanded-by-default would create visual noise.

## Interaction Contract

### Toggle Button

- Click: `setFiltreVisible(v => !v)` — immediate, no animation delay
- `style={{ touchAction: 'manipulation' }}` — prevents 300ms tap delay on mobile
- Badge: rendered as `<span className="bg-brand-primary text-white rounded-full px-1.5">{nrFiltreActive}</span>` — inline in button text, visible even when panel is collapsed
- Label: `"Filtrează"` (zero active) / `"Filtrează (N)"` (N active)

### Reset Link

- Visible: only when `nrFiltreActive > 0`
- Click: calls `resetFiltre()` prop — no confirmation dialog, no undo
- Placement: inline in same row as toggle button, `text-xs text-slate-400 hover:text-white underline`

### Gen Chips

- Each chip is a `<label>` with hidden `<input type="checkbox">` inside
- Checked state driven by `filtre.gen.has(gen)` — controlled
- onChange: `toggleGen(gen)` prop
- Values: `['Feminin', 'Masculin', 'Mixt']` — hardcoded, not from props (these are domain constants)
- `className="cursor-pointer"` on label

### Probă Select

- Native `<select>` — consistent with rest of codebase
- `value={filtre.probaId}` controlled
- onChange: `e => setFiltre({ probaId: e.target.value })`
- First option: `<option value="">Toate probele</option>` — always present

### Vârstă Inputs

- Two `<input type="number" min={0}>` side by side
- `placeholder="Min"` / `placeholder="Max"`
- Width: `w-20` (80px each)
- onChange: `e => setFiltre({ varstaMin: e.target.value })` / `varstaMax`
- Separator: `<span className="text-slate-500 text-xs">–</span>`

### Grad Selects

- Two native `<select>` side by side (min grad / max grad)
- `value={filtre.gradMin}` / `value={filtre.gradMax}` — value is `grad.ordine.toString()`, NOT `grad.id`
- First option: `<option value="">Orice grad</option>` in each
- Options: `[...grade].sort((a, b) => a.ordine - b.ordine)` — sorted ascending
- `<option value={g.ordine.toString()}>{g.nume}</option>` — user sees grad name, not ordine number
- Separator: `<span className="text-slate-500 text-xs">–</span>` between the two selects

**Critical:** `value` must be `ordine` (string), never `id`. `aplicaFiltreCategorie` compares against `cat.grad_min_ordine` (numeric field), not FK id.

**Source:** RESEARCH.md Pitfall 5 [VERIFIED: types.ts `CategorieCompetitie.grad_min_ordine: number | null`].

## Visual Order of Filter Groups

Left-to-right / top-to-bottom reading order within the 2-column grid:

1. **Gen** (top-left) — most-used filter, highest priority
2. **Probă** (top-right) — sport-specific, second priority
3. **Vârstă** (bottom-left) — numeric range
4. **Grad** (bottom-right) — domain-specific, requires grade knowledge

**Rationale:** Mirrors the column order in category tables (gen → probă → vârstă → grad). User mental model matches the list they are filtering.

**Source:** Claude's Discretion (CONTEXT.md) — decision made here based on domain priority.

## Accessibility Contract

- All `<input type="checkbox">` chips: wrapped in `<label>` — click target is the full chip, not just the hidden checkbox
- All `<input>` and `<select>` fields: section header text serves as visual label (no `<label for>` required at this UX tier — compact inline filter bar)
- Toggle button: uses `<button>` element — keyboard focusable, enters via Tab
- `touchAction: 'manipulation'` on all interactive elements in filter bar — prevents 300ms mobile delay
- Color contrast: inactive chips `text-slate-300` on `bg-slate-700/50` — meets WCAG AA at this font size
- Focus ring: `focus:outline-none focus:border-brand-primary/60` — visible blue border on focused inputs

## Phase 5 Color System Integration

Phase 5 deployed 12 CSS variables with `--t-` prefix into `index.css :root`. This phase uses:

| CSS Variable | Phase 5 Default | Usage in FilterBar |
|---|---|---|
| `--t-primary` | #3b82f6 | Active chip + toggle button background; badge; checkbox accent |
| `--t-surface` | #0f172a | Inherited page background |
| `--t-surface-2` | #1e293b | Filter panel background (`bg-slate-800/60` references this tonally) |
| `--t-border` | #1e293b | Filter panel border; input borders |
| `--t-text` | #f8fafc | Input text; select text |
| `--t-text-muted` | #94a3b8 | Section labels; inactive chip text |

**Note:** The Tailwind classes `bg-brand-primary/20`, `border-brand-primary/50`, `text-brand-primary`
reference `--brand-primary` (legacy CSS var = same #3b82f6 value as `--t-primary`). These classes
are pre-verified in `InscrieriView.tsx`. Do NOT replace them with `var(--t-primary)` inline styles —
keep using the Tailwind classes for consistency with the existing InscrieriView implementation.

When a user switches to a non-blue theme (Phase 5 ThemeEditor), `--t-primary` changes but `--brand-primary`
does NOT (legacy variable). Phase 7 integration note: if full theme-responsiveness is desired for
filter active states, Phase 7 should replace `bg-brand-primary/*` classes with `style={{ backgroundColor: 'var(--t-primary)' }}` inline styles. This is **out of scope for Phase 6**.
