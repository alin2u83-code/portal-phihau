## Spacing Scale

Declared values (multiples of 4 only). Source: Tailwind defaults + existing patterns in `GrupaDetailView.tsx` and `AdaugaSportiviModal.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), dot gaps (`gap-0.5`), badge padding (`py-0.5`) |
| sm | 8px | Compact element spacing (`gap-2`), panel row padding (`py-2 px-3`) |
| md | 16px | Default element spacing (`gap-4`), header padding (`px-4`) |
| lg | 24px | Section padding (`py-6`), calendar cell padding |
| xl | 32px | Layout gaps (`py-8`) |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Calendar cell touch target: min 40px height (aligns with `min-h-[40px]` pattern from TabSportivi buttons)
- Day panel row min height: 44px (touch-friendly action buttons per mobile requirements)
- Dot size: 6px diameter (`w-1.5 h-1.5`) — below scale, acceptable for decorative indicator only

## Layout Contract

### Calendar Grid (D-01, D-02)

```
[CalendarHeader]
  ChevronLeft  |  "Lună An" (Heading 16px bold)  |  ChevronRight
  [Week header row: L M M J V S D — Label 12px semibold slate-400]
  [Grid cells: 7 columns × 4-6 rows]
    [Cell]
      [Day number: 14px, bold if today]
      [Dot row: max 3 dots (6px) + "..." if overflow]
```

- Grid: `grid grid-cols-7` with Tailwind div — no external calendar library
- Cell min height: 40px, cursor-pointer
- Empty cells (offset days): no number, no dots, no hover state
- Cell click → `setSelectedDate(dateStr)`, panel below updates

### DayPanel (D-05, D-06, D-07)

```
[DayPanel — appears below grid when selectedDate !== null]
  [Header: "Antrenamente — [zi, data]" — 14px slate-300]
  [IF no antrenamente for day]
    "Niciun antrenament pe [data formatată]"  +  Button "Adaugă Antrenament"
  [IF antrenamente exist]
    [Per antrenament row — flex items-center, min 44px, rounded-lg bg-slate-800/30 border-slate-700/50]
      [Time: "HH:MM–HH:MM" font-mono 14px slate-200]
      [Badge status]
      [Actions — ml-auto]
        Button "Anulează" (if not anulat)
        Button "Reactivează" (if anulat)
        ConfirmButton "Șterge" (always)
```

- Panel is a fixed DOM element below grid — not modal, not inline row expand
- No animation on open (performance — `animate-slide-up` optional, matches `index.css`)

### ModalAdaugare (D-08)

```
[Modal title: "Adaugă Antrenament"]
  [Input: Data — type="date", pre-filled selectedDate || todayLocal]
  [Input: Ora start — type="time", default "18:00"]
  [Input: Ora sfarsit — type="time", default "19:30"]
  [Footer: Button "Salvează" (primary) | Button "Anulează" (secondary)]
```

### ModalAnulare (D-10)

```
[Modal title: "Anulare Antrenament"]
  [Textarea: "Motiv anulare (opțional)" — resize-none, 3 rows]
  [Note: "Antrenamentul va rămâne în calendar cu status anulat."]
  [Footer: Button "Anulează Antrenament" (danger) | Button "Renunță" (secondary)]
```

## Accessibility Contract

- All interactive calendar cells: `cursor-pointer` + focusable via keyboard (tabIndex implied by div onClick)
- Buttons in DayPanel: minimum 40px height, `touch-manipulation` class for mobile
- Modal uses `Modal` from `components/ui.tsx` which handles Escape key, focus trap, portal in `document.body`
- Icon-only buttons (ConfirmButton with TrashIcon only): must have accessible label via title or aria-label: `aria-label="Șterge antrenament"`
- Badge text is visible in all badge variants (color-mix ensures sufficient contrast against dark background)
- `font-mono` for time display ensures consistent character width alignment in rows
